## 一、我能實作的部分

### 1.1 核心演算法（可獨立完成）

| 模組 | 說明 | 難度 |
|------|------|------|
| **大衍之數運算** | 49 根蓍草 → 分二、掛一、揲四、歸奇 → 三變一爻 → 6/7/8/9 對應老陰/少陽/少陰/老陽 | 低 |
| **本卦生成** | 18 變得出 6 爻，由下往上組成卦象 | 低 |
| **變卦計算** | 老陰(6)→陽、老陽(9)→陰，其餘不變 | 低 |
| **互卦計算** | 取本卦 2、3、4 爻為下卦，3、4、5 爻為上卦 | 低 |
| **卦象編碼** | 64 卦的二進位/十進位對應（如 111111=乾、000000=坤） | 低 |

大衍之數的數學邏輯是固定的，無需額外「算法參考」。


若你提供一份 64 卦的 JSON 範例（至少 2–3 卦完整結構），我可以依此設計 schema 並擴充為 64 卦。

## 三、分階段實作建議

### 階段一：MVP（約 1–2 週）

1. **大衍之數核心**：`js/calc/dayanDivination.js` — 純運算，無 UI
2. **64 卦查表**：`data/iching/hexagrams.json` — 需你提供或我找公開資料
3. **簡易占卦頁**：`divination.html` — 問題輸入、18 次點擊、顯示本卦+變卦+互卦+卦辭爻辭
4. **存庫**：`divination_logs` 表，記錄問題、卦象、時間

**產出**：可用的占卦功能，解釋以卦辭爻辭為主，尚無五行流年。

### 階段二：五行流年整合（約 1 週）

1. 卦宮五行 + 流年干支（從 `baziCore` / 現有流年邏輯取得）
2. 已登入用戶：讀取 `user_charts` 的八字，計算喜用神
3. 產出「卦象 × 流年 × 喜用神」的簡要建議（需你提供或確認文案規則）

### 階段三：營運功能（依需求）

1. 每日一占、分享卦象圖片
2. 7 天後推播「上週占的某某事，進展如何？」
3. 五行流年深度解析 / 大師詳解 → 積分或訂閱解鎖
4. 高頻詞分析 → 人生轉折報告
