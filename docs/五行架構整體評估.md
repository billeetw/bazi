# 八字五行戰力架構 — 整體評估

從準確度、底層邏輯聯動、數值與邏輯正確性、東方哲學匹配度、科學性五個維度進行分析。

---

## 一、準確度 (Accuracy)

### 1.1 傳統八字論命對照

| 面向 | 現況 | 傳統子平 | 評述 |
|------|------|----------|------|
| 五行來源 | 天干 + 地支藏干 | ✓ 一致 | 藏干權重採主流說法（本氣／中氣／餘氣），可追溯 |
| 月令 | 僅月支依季節乘數 | 月令為提綱，影響最大 | ✓ 符合「月令為綱」 |
| 日主 | 未單獨加權 | 日干為我，多數派以之為主 | ⚠ 實戰結構未顯式區分日主，採整體向量 |
| 十神 | 日干 vs 月干 | 日干 vs 各干 | ✓ 取月干合理，因月令影響最大 |
| 流年/流月 | 未納入 | 大運／流年會變 | 設計為靜態本命盤，符合目前範圍 |

### 1.2 數值語意

- **表層 (surface)**：表層結構 = 天干 + 藏干，語意清楚。
- **實戰 (strategic)**：語意為「在月令影響下的實際力量」。
- **限制**：無大運／流年，僅為本命盤靜態分析。

**準確度小結**：在「本命盤靜態五行分析」前提下，架構與主流子平、五行理論對齊良好；若擴充流年／流月，需再定義與實作。

## 二、底層邏輯聯動 (Logic Coherence)

### 2.1 資料流

```
四柱(bazi.display) 
  → buildPillarsFromBazi 
    → vectorize (天干+藏干) → v_raw
    → applySeason (月令)     → v_season
      → buildReport (五行生剋報告)
      → surface = v_raw, strategic = v_season
        → renderBar / getPoYenBoard / generateFiveElementComment
        → toEnergyLevelsFromWx (0-3 等級)
          → relationBadge / meaningText / 風險/建議
```

**優點**：
- 單向、明確：四柱 → 數值 → 報告 → UI。
- 表層／實戰共用同一套數值流程，僅在 season 是否套用上有差異。
- 五行生剋 (gen_edges、ctl_edges) 與報告生成、PoYen 看板一致。

### 2.2 聯動是否完整

| 模組 | 輸入 | 輸出 | 聯動 |
|------|------|------|------|
| WuxingFlowPipeline | 四柱、KB | v_raw, v_season, report | ✓ |
| getPoYenBoard | surface/strategic | 核心結構、轉換、風險、助推 | ✓ |
| YinyangAdvanced | yangPct, tenGod | 年度主軸、交叉洞察 | ✓ |
| WuxingOneLiner | strategic | 陰陽%、一句註解 | ✓ |

**待確認**：
- 陰陽計算：`yang = 木+火, yin = 土+金+水` 為簡化版，傳統有不同劃分方式；若無特別需求，目前設定可接受。
- 能量等級 (toEnergyLevelsFromWx)：用 `v/max` 分檔 (0.25/0.5/0.75)，屬經驗規則，與傳統派別無直接對應，但內部邏輯一致。

## 三、數值與邏輯正確性 (Correctness)

### 3.1 公式檢查

- **vectorize**：天干 +1、藏干 × weight，邏輯正確。
- **applySeason**：`v_season[e] = v_raw[e] × m[e]`，乘法律一致。
- **相生 throughput**：`min(上游, 下游)`，符合「瓶頸在弱端」。
- **相剋壓力**：`ctlVal/toVal` 作為 control_ratio，合理。

### 3.2 邊界情況

| 情況 | 處理 | 評價 |
|------|------|------|
| 全 0 | Fallback、預設值 | ✓ |
| 缺少時柱 | hG/hZ 可為空，不影響 | ✓ |
| KB 404 | Fallback 只用天干 | ✓ |
| 最強/最弱相同 | getStrongestWeakest 用 first/last | ⚠ 全同分時語意略模糊，可接受 |

### 3.3 潛在風險

1. **藏干權重**：branches.json 權重 (0.6/0.3/0.1 等) 採常見配置，若採用不同流派需調整。
2. **月令乘數**：1.3 / 0.9 為設計參數，缺乏古籍定量依據，屬實務折衷。
3. **levelOf 區間**：0.25/0.5/0.75 為固定門檻，對極端分布可能略粗糙。

## 四、東方哲學匹配度 (Philosophy Alignment)

### 4.1 五行生剋

- **相生**：水生木→木生火→火生土→土生金→金生水，與傳統一致。
- **相剋**：木剋土→土剋水→水剋火→火剋金→金剋木，與傳統一致。
- **elements.json**：gen/ctl 邊定義與上述關係一致。

### 4.2 月令與節氣

- 寅卯木、巳午火、辰未戌丑土、申酉金、亥子水，符合傳統月令五行。
- 乘數設計反映「當令者旺、失令者衰」的觀念。

### 4.3 十神

- 比肩、劫財、食神、傷官、正財、偏財、正官、七殺、正印、偏印，十神完整。
- tenGodFromStems 依日干與他干之五行、陰陽關係推導，符合子平邏輯。

### 4.4 陰陽

- 陽干：甲丙戊庚壬；陰干：乙丁己辛癸。
- 陰陽比例簡化為「木火 vs 土金水」，與常見「陽動／陰靜」詮釋近似。

### 4.5 結構語意

- 表層 ≈ 表面氣勢、名望（天干＋藏干）。
- 實戰 ≈ 實際實力、勝負（再乘月令）。
- 語意與傳統「得令／失令」觀念對應良好。

## 五、科學性 (Scientific Rigor)

### 5.1 可重現性

- 相同四柱 → 相同 v_raw、v_season、報告。
- 計算為純函數，無隨機性。
- KB 以 JSON 明確定義，可版本管理。

### 5.2 可驗證性

- 公式透明，可人工逐項驗算。
- 與他種八字軟體比對，可檢驗藏干、月令是否一致。

### 5.3 可反證性 (Falsifiability)

- 若「某人五行分布應為 X」有明確定義，可設計測試案例驗證。
- 傳統論命多為定性描述，定量門檻需自行定義。

### 5.4 限制與邊界

- 本架構為**本命盤靜態**分析，不處理大運、流年、流月。
- 月令乘數、藏干權重、能量等級區間為**設計參數**，非來自單一古籍或實驗。
- 與實證科學不同：八字屬文化／詮釋體系，本質上難以用統計顯著性全面驗證。

## 七、建議改進方向（若未來擴充）

1. **日主加權**：在 strategic 中對日干所屬五行額外加權，更貼近「日主為我」。
2. **藏干流派**：支援多套 branches.json（如《三命通會》 vs 現代簡化版）供選擇。
3. **月令參數化**：將 1.3 / 0.9 等抽成設定，便於與不同派別對照。
4. **能量等級可調**：levelOf 的 0.25/0.5/0.75 改為可配置。
5. **流年整合**：若有流年干支，可定義「本命 + 流年」的疊加與權重規則。
6. **AB 測試**：蒐集用戶回饋，對照「建議補土／補木」等是否符合主觀體驗，用於微調文案與門檻。
