# 占卦能力邊界與優化基礎

供後續優化決策參考。

---

## 一、卦怎麼來：隨機？時間種子？使用者選項？

### 現況

| 項目 | 實作 | 說明 |
|------|------|------|
| **隨機源** | `Math.random()` | `dayanDivination.js` 第 11 行：`Math.floor(Math.random() * (stalks - 2)) + 1` |
| **時間種子** | ❌ 無 | 未使用 `Date.now()`、timestamp 或 crypto 作為種子 |
| **使用者選項** | ❌ 無 | 無「真隨機 / 時間種子 / 手動選爻」等選項 |

### 流程

1. 使用者寫問題、選情緒（焦慮/期待/平靜）、選時間維度（6 月 / 1 年 / 6 年 / 階段）
2. 長按中央區域，每 1200ms 自動產生一爻（共 6 爻）
3. 每爻呼叫 `DayanDivination.getOneLine(49)`，內部用 `Math.random()` 模擬「分二、掛一、揲四、歸奇」

### 優化方向

| 選項 | 優點 | 缺點 |
|------|------|------|
| **維持 Math.random** | 簡單、無狀態 | 可被預測、無「當下時刻」連結 |
| **時間種子** | 可重現、有「問卦當下」感 | 需決定種子粒度（秒/毫秒） |
| **crypto.getRandomValues** | 較安全、較難預測 | 需 polyfill、略增複雜度 |
| **使用者選項** | 可滿足不同偏好 | 增加 UI、需說明差異 |

## 二、解釋來源：自寫規則 / 資料庫 / 模型生成 / 混合？

### 現況：混合（規則 + 靜態資料）

| 來源 | 檔案 / 模組 | 內容 |
|------|-------------|------|
| **靜態資料庫** | `data/iching/*.json` | 64 卦卦辭、爻辭、卦宮五行、384 爻參數 |
| **自寫規則** | `divinationInterpretation.js` | 互卦/變卦解釋、情境覆寫（事業/財運/感情） |
| **自寫規則** | `divinationInsight.js` | 64 卦 × 情境關鍵字矩陣、當位/應與形容詞 |
| **自寫規則** | `scenarioEngine.js` | 五行 × 情境矩陣、爻位 × 情境階段、流年加權 |
| **自寫規則** | `divinationScore.js` | 384 爻加權：base + correct + resonance × yearWeight |
| **自寫規則** | `divinationWuxing.js` | 卦宮五行 × 2026 流年、方位規則 |
| **自寫規則** | `divinationReminders.js` | 負分月預警、動爻月提醒文案模板 |
| **模型生成** | ❌ 無 | 無 LLM / AI 生成解釋 |

### 資料檔一覽

| 檔案 | 大小 | 用途 |
|------|------|------|
| hexagrams.json | 24K | 64 卦卦名、卦辭、六爻爻辭 |
| hexagram-palace.json | 4K | 卦 → 卦宮、卦宮 → 五行 |
| hexagram-summaries.json | 8K | 卦辭白話摘要、卦性格 |
| hexagram-commentaries.json | 20K | 彖傳、大象傳 |
| hexagram-structure.json | 20K | 上下卦、結構說明、矛盾、學習提示 |
| hexagram-opening-wisdom.json | 12K | 開竅智慧、修養核心 |
| hexagram-actions.json | 8K | 學習任務 |
| lines-384.json | 96K | 384 爻 base/correct/resonance/mag/hint（腳本生成） |

### 解釋流程（簡化）

```
本卦 primaryIndex → hexagrams.json 卦辭
                  → hexagram-palace 卦宮五行
                  → divinationInterpretation 大局解釋（含情境覆寫）
                  → divinationInsight 情境關鍵字

動爻 → lines-384.json 當位/應與/爻辭 hint
    → divinationScore 月分數
    → divinationReminders 預警文案

互卦/變卦 → divinationInterpretation 互卦/變卦解釋

流年 → divinationWuxing 2026 丙午年五行關係、方位
```

## 三、玄 vs 務實：你希望保持多玄、多務實？

### 現況定位

| 面向 | 現況 | 說明 |
|------|------|------|
| **儀式感** | 中等 | 長按蓄力、蓍草動畫、五行背景切換、ceremony 結果頁 |
| **術語** | 偏傳統 | 卦辭爻辭原文、當位/應與、卦宮五行、彖傳大象傳 |
| **務實** | 中等 | 事業/財運/感情情境、月分數趨勢、轉折預警、2026 方位 |
| **文案風格** | 混合 | 「卦象提醒」「宜守不宜進」+ 具體行動建議（學習任務、方位） |

### 可調維度

| 維度 | 更玄 | 更務實 |
|------|------|--------|
| **卦辭呈現** | 保留原文、彖象傳 | 只給白話摘要、行動建議 |
| **解釋深度** | 多談「卦象格局」「天意」 | 多談「可執行步驟」「時間點」 |
| **分數/趨勢** | 隱藏或弱化 | 強化月分數、轉折提醒 |
| **情境** | 少情境、重整體 | 多情境（事業/財運/感情/健康） |
| **流年** | 抽象五行生剋 | 具體月份建議、方位運用 |

## 四、產卦與解卦的能力邊界

### 產卦（目前）

- ✅ 大衍之數 18 變、三變一爻
- ✅ 本卦、變卦、互卦、動爻
- ✅ 長按 UX 模擬揲蓍
- ❌ 無時間種子、無使用者選項
- ❌ 無「手動選爻」模式

### 解卦（目前）

- ✅ 64 卦卦辭、爻辭、卦宮五行
- ✅ 本卦/互卦/變卦解釋
- ✅ 事業/財運/感情情境
- ✅ 384 爻當位/應與、月分數、動爻解析
- ✅ 2026 流年五行、方位
- ❌ 無 LLM 生成
- ❌ 無個人化（未結合八字/紫微）
- ❌ 問題內容不影響解釋（僅供儲存與紀錄）

### 優化可選方向

1. **產卦**：加入時間種子或 crypto 隨機、可選「簡易模式」
2. **解卦**：問題關鍵字 → 情境推測、或 LLM 生成客製化摘要
3. **個人化**：登入用戶可帶入八字五行，解卦時加權
4. **玄/務實**：依產品定位調整文案與功能比重
