# 架構優化建議

## 現況摘要

| 項目 | 數值 |
|------|------|
| index.html | 2,523 行、~111 KB |
| 內嵌 CSS | ~1,630 行（約 65%） |
| 內嵌 script | 6+ 區塊、~170 行 |
| dist/app.js | ~392 KB (gzip ~127 KB) |

---

## 1. 抽離 CSS（優先、效益最高）

**現況**：index.html 內含 1,600+ 行 `<style>`，約佔 65%。

**建議**：
- 建立 `css/main.css`，把內嵌樣式移出
- 在 `<head>` 改為 `<link href="css/main.css" rel="stylesheet">`
- 可再加 build 步驟 minify（可節省 20–30%）

**效益**：
- HTML 縮小 ~60%
- CSS 可獨立 cache，第二次造訪更少重載
- 維護、搜尋、重構較容易

---

## 2. 整合內嵌 scripts

**現況**：頁尾有多段 inline script：
- VH fallback + LINE 偵測
- i18n（applyDataI18n、語系切換、表單還原）
- AuthService.init（已改由 app.js 負責，可移除）
- 手機選單開關
- startup-sequence 動態載入

**建議**：
- 合併成一個 `dist/bootstrap.js`（或單一 inline block）
- 移除重複的 `AuthService.init()` 呼叫
- 視情況把 VH、LINE、語系等邏輯整進 app bundle

---

## 3. Tailwind 與樣式策略

**現況**：使用 Tailwind CDN，載入完整套件。

**建議**（可選）：
- 正式環境改用 Tailwind build，只輸出實際使用的 utilities
- 預期可減少 100KB+ 的 CSS 下載量

---

## 4. app.js 程式碼分割（可選）

**現況**：單一 392KB 的 app bundle。

**建議**：
- 拆出較少用的功能（如 expert-questionnaire、birth-time-identifier）做 lazy load
- 主流程首次載入只載核心 UI + 計算與命盤區塊

---

## 5. 其他整理

| 項目 | 建議 |
|------|------|
| CEREMONY_FALLBACK 等大型物件 | 從 ui.js 移至獨立模組或 JSON |
| 文章列表 | 若結構穩定，可考慮模板或 SSG |
| 重複 UI 邏輯 | 手機選單開關可抽成共用元件 |
| 未使用檔案 | 檢查 index3.html、test-*.html、admin-copy.html 是否仍需保留 |

### 關於 Code Splitting

- 目前使用 IIFE 格式，Rollup 不支援 IIFE + code splitting
- 若需 lazy load，需改為 `format: "es"` 並在 index.html 使用 `<script type="module" src="dist/app.js">`
