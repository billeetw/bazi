# 諮詢表單確認信實作計畫

> 目標：報名送出後，寄送確認信給用戶（含預約摘要），並顯示「預計 1–2 工作天內回覆」。

---

## 一、現況

| 項目 | 狀態 |
|------|------|
| 表單送出 | ✅ 寫入 D1 `consultations` |
| 成功畫面 | ✅ Modal + Calendly 連結 |
| 確認信 | ❌ 無 |
| 預計回覆時間提示 | ❌ 無 |

---

## 二、實作項目

### 2.1 前端：成功訊息優化（已實作）

- 在成功 Modal 與 inline 訊息加入：「我們將在 **1–2 個工作天內** 與您聯繫確認時段。」
- 讓用戶知道何時會收到回覆。

### 2.2 後端：寄送確認信

**流程**：`/api/consultation` 成功寫入 DB 後 → 呼叫 Resend API 寄信給用戶。

**發信服務**：Resend（https://resend.com）

- 免費額度：每月 3,000 封
- 支援自訂網域
- 簡單 REST API，Cloudflare Workers 可用 `fetch` 呼叫

---

## 三、Resend 設定步驟（需你執行）

### 步驟 1：註冊 Resend

1. 前往 https://resend.com 註冊
2. 驗證 Email

### 步驟 2：新增網域

1. Resend Dashboard → **Domains** → **Add Domain**
2. 輸入 `17gonplay.com`（或發信用的子網域如 `mail.17gonplay.com`）
3. 依指示在 Cloudflare DNS 新增 Resend 提供的 SPF、DKIM 等紀錄
4. 驗證完成後，發信地址可用 `noreply@17gonplay.com` 或 `consult@17gonplay.com`

### 步驟 3：建立 API Key

1. Resend Dashboard → **API Keys** → **Create API Key**
2. 複製產生的 key（格式 `re_xxxxx`，只顯示一次）
3. 在 Cloudflare Pages 專案：
   - **Settings** → **Environment variables**
   - 新增 `RESEND_API_KEY` = 你的 key
   - 套用到 **Production**（與 Preview 若需要）

---

## 四、程式碼變更

### 4.1 `functions/api/consultation.js`

- 在 `return new Response(JSON.stringify({ ok: true, id }))` 之前
- 呼叫 `sendConfirmationEmail(env, { name, email, birth, topics, topicsString })`
- 發信失敗不影響 API 成功（fire-and-forget，或可 log 錯誤）

### 4.2 確認信內容（HTML）

- 主旨：`【人生說明書】諮詢報名確認 - 我們已收到您的資料`
- 內文：
  - 稱呼（姓名）
  - 報名摘要：出生資訊、想討論的主題
  - 下一步：請到 Calendly 選擇時段
  - 預計 1–2 工作天內會與您聯繫確認
  - 聯絡方式（若有）

---

## 五、環境變數檢查清單

| 變數 | 用途 | 設定位置 |
|------|------|----------|
| `RESEND_API_KEY` | Resend 發信 | Pages → Settings → Environment variables |
| `CONSULT_FROM_EMAIL` | 發信地址（選填） | 預設 `noreply@17gonplay.com`；測試期可設 `onboarding@resend.dev` |
| `CONSULT_DB` | 已設定 | wrangler.toml |

若未設定 `RESEND_API_KEY`，API 仍正常寫入 DB，僅不寄信。

## 六、測試
