# 推算時辰故障排查

「點我推算時辰」涉及：登入 → 開問卷 → 提交 → `POST /api/me/estimate-hour`。

## 1. 問題定位

請在瀏覽器 Console（F12）執行，回傳結果：

```javascript
// 1. Config API 是否正常
fetch('/api/auth/config').then(r=>r.json()).then(console.log)

// 2. 若已登入，測試 estimate-hour（需先登入）
fetch('/api/me/estimate-hour', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', ...(window.AuthService?.getAuthHeaders?.()||{}) },
  body: JSON.stringify({ answers: { q1:['A'], q2:['A'], q3:'A', q4:'A', q5:'A', q6:'A', q7:'A', q8:'A', q9:'A', q10:'A', q11:'A', q12:'A', q13:'A', q14:'A', q15:'A', q16:'A', q17:'A', q18:'A', q19:'A' } })
}).then(r=>r.text()).then(console.log)
```

## 2. Cloudflare Pages 設定檢查

**Cloudflare Dashboard → Workers & Pages → 你的 Pages 專案 → Settings**

### 2.1 Functions 綁定
- **Settings → Functions → D1 database bindings**
- 需有 `CONSULT_DB` 綁到 `consult-db`

### 2.2 Environment variables（Production）
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`
- `JWT_SECRET`（至少 32 字元）

### 2.3 若曾改過 Cloudflare 設定
- 檢查是否有新建立的 Pages 專案、Worker 覆蓋了舊的
- 確認 `wrangler pages deploy .` 部署的是正確專案（`wrangler.toml` 的 `name = "bazi"`）

## 3. 常見症狀

| 症狀 | 可能原因 |
|------|----------|
| 點擊後出現「登入功能尚未設定」 | Config 未載入或 GOOGLE_CLIENT_ID 未設 |
| 點擊後 Google 登入開不了 / 失敗 | GOOGLE_CLIENT_SECRET 或 callback URL 有誤 |
| 問卷可開、提交後「推算失敗」 | JWT 過期、D1 未綁定、或 DB schema 未跑 migration |
| 提交回 401 | 未登入或 token 過期 |
| 提交回 500 | 檢查 Pages 後端日誌（Real-time Logs） |

## 4. D1 與 Migrations

`/api/me/estimate-hour` 需寫入 `estimate_hour_logs`。確認 D1 `consult-db` 已跑 migrations：

```bash
# 本機對 consult-db 執行 migrations
npx wrangler d1 execute consult-db --remote --file=./migrations/0008_estimate_hour_logs.sql
npx wrangler d1 execute consult-db --remote --file=./migrations/0009_estimate_hour_logs_add_user_ip.sql
npx wrangler d1 execute consult-db --remote --file=./migrations/0010_estimate_hour_logs_vnext.sql
```

（需先在 wrangler.toml 指定正確的 `database_id`）
