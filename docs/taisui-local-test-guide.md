# 太歲光明燈 — 本地測試指南

## 一、會出現在哪裡？

| 項目 | 位置 | 出現時機 |
|------|------|----------|
| **太歲卡片** | 紫微 12 宮區塊**下方**、八字五行區塊**上方** | 計算完成後 |
| **導覽「太歲」** | 頂部 nav（紫微、太歲、八字、流年） | 進入儀表板後 |
| **點光明燈按鈕** | 太歲卡片內 | 已登入 + 計算完成 |
| **光明燈 Modal** | 點擊「點光明燈」後，畫面中央彈出 | 已登入 + 點擊按鈕 |
| **光明燈勳章** | 頂部「登入為 xxx」旁 | 已登入 + 已點燈 |

## 二、必做步驟（依序）

### Step 1：確認 build 已更新

```bash
npm run build:main
```

### Step 2：啟動開發伺服器

```bash
npx wrangler pages dev . --port 8788
```

或：

```bash
./start-dev.sh 8788
```

### Step 3：開啟瀏覽器

前往：**http://localhost:8788/**

### Step 4：輸入出生資料

- 年、月、日（必填）
- 時辰或時分（選一種）
- 性別

### Step 5：點「開始人生分析」

- 會出現載入動畫（約 3–5 秒）
- 計算完成後，輸入區會隱藏，儀表板會顯示

### Step 6：找到太歲區塊

- **向下捲動**：太歲在紫微 12 宮下方
- 或點頂部導覽的 **「太歲」** 跳到該區

### Step 7：點燈（兩種流程）

**流程 A：未登入時直接點燈（2A 暫存流程）**

- 計算完成後，在太歲區點「**登入並點光明燈**」
- 系統會暫存你的出生資料到 sessionStorage，並開啟 Google 登入
- 登入成功後會自動：儲存命盤 → 點燈 → 顯示 Modal 動畫與守護語
- 關閉分頁後暫存會清除（sessionStorage）

**流程 B：已登入時點燈**

- 先登入，再輸入出生資料並計算
- 若尚無命盤：點「點光明燈」時會自動儲存當前命盤後再點燈
- 若已有命盤：直接點燈即可

## 三、常見問題排查

### 1. 只看到「計算完成後顯示」，沒有太歲內容

**可能原因**：API 失敗或尚未完成計算

- 打開開發者工具（F12）→ Console，看是否有錯誤
- Network 檢查 `/api/taisui/status` 是否回 200
- 確認已點「開始人生分析」且計算成功（紫微、八字有內容）

### 2. 完全看不到太歲區塊

**可能原因**：尚未進入儀表板

- 太歲在 `#system` 內，只有計算完成後才會顯示
- 確認：輸入區已隱藏、能看到紫微 12 宮
- 若看到紫微，向下捲動即可看到太歲

### 3. 點光明燈沒反應

**可能原因**：未登入或沒有出生資料

- 未登入：點「登入並點光明燈」→ 需先完成計算（有出生資料才會暫存）
- 已登入無命盤：點「點光明燈」時會自動儲存當前表單資料後再點燈
- 若表單為空（例如直接進入頁面未計算）：會提示「請先輸入出生資料並計算」

### 4. 點燈後 Modal 沒出現

- 檢查 Console 是否有錯誤
- 檢查 `/api/taisui/lamp` 是否回 200
- 若回 400「請先建立命盤資料」：系統會嘗試自動儲存表單資料後重試；若表單為空則需先計算

### 5. dist/app.js 可能過期

若剛改過程式，請重新 build：

```bash
npm run build:main
```

然後重新整理瀏覽器（必要時強制重新載入 Ctrl+Shift+R）。
