## 四、階段規劃（重做順序）

### 第一階段：時間軸四項基礎「只出不錯」✅ 已完成

- **目標**：同一份命盤，大限／流年不互相套錯，且只從「單一真值來源」輸出。
- **產出**：
  1. **大限命宮**：當前大限宮位（如僕役宮）。
  2. **大限四化**：當前大限天干化出的四顆星（廉貞化祿、破軍化權、武曲化科、太陽化忌），**不得**出現流年四化星（天機、文昌等）。
  3. **流年命宮**：當年流年宮位（如疾厄宮）。
  4. **流年四化**：當年流年天干化出的四顆星（天同、天機、文昌、廉貞等）。
- **已做**：
  - **sihuaTimeBuilders.ts**：大限四化改為**優先 decadalLimits[當前].mutagenStars**，缺項才用 overlap 的「大限」層補；禁止使用 liunian。流年四化註明僅來自 liunian / fourTransformations.liunian，禁止 decadal。
  - **lifebookSection-zh-TW.json**：s17「十年×年度交叉」改為一句導向 s15，不再重複內容；s15a 改為一句導向 s15，避免空佔位符整段輸出。
- **驗收**：時間軸校對總覽中，大限四化與流年四化星名不同（且大限四化符合該大限天干）。需以實際命盤／資料庫版再跑一次確認。

### 第二階段：飛星路徑與解釋（技術版 + 解釋段）

- **目標**：四化飛星技術版 4 條齊全（祿權科忌各一），且每條的「解釋段」與技術版星+化一致。
- **產出**：
  - 【四化飛星技術版】大限 4 條：每條「星化X 自 A 宮出 飛入 B 宮」。
  - 每條對應一段「星象徵 + 化X帶來 + 代表從A到B + 因此…」，且該段內的「化X」與技術版一致（不可出現技術版化祿、解釋寫化科）。
- **做法**：
  - 技術版與解釋段共用同一 list（同一批 FourTransformLine），依序組裝；避免技術版用一套、解釋用另一套。
  - 若解釋段目前是「第一條用祿、內容卻從科來」的 bug，改為「第 i 條用第 i 筆的 star + type + from + to」。
- **驗收**：同一星同一化（如廉貞化祿）在技術版與解釋段一致；4 條都出現。

### 第三階段：落宮／能量集中／本命宮干飛化區塊

- **目標**：【本命／大限／流年四化落宮】、【四化能量集中】、【本命宮干飛化】有內容且不空。
- **做法**：確認 build 函數（如 buildSihuaFallByPalaceBlock、sihuaEnergyFocusBlock、natalGongganFlowBlock）有被呼叫，且其結果寫入 findings / placeholder map 並對應到正確 template 佔位符。

### 第四階段：主戰場、關鍵年份、決策建議

- **目標**：主戰場／次戰場、十年×年度交叉、關鍵年份、現在／一年內／這十年 有合理內容或明確 fallback。
- **做法**：盤點 s15MainBattlefield、key years、yearRoleInDecade 等欄位來源，接上 overlap 或既有評分／標籤，並處理空值時的 fallback 文案。

### 第五階段：s18/s19/s21 與除錯輸出

- **目標**：主要盲點、行動策略、靈魂總結 不再出現「此宮暫無星曜與四化資料」的錯誤提示；FLOW_DEBUG 不出現在正式命書。
- **做法**：釐清 s18/s19/s21 的「宮位」語意（是否為「當前流年宮」或「全盤」）；若為全盤總結，不應綁單宮。FLOW_DEBUG 僅在技術/除錯模式或開發環境輸出。
