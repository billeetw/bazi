# 流月戰略與紫微五行比例顯示檢查

## 可能原因與檢查點

### 1. 流月戰略（monthGrid）不顯示

| 檢查項 | 說明 |
|--------|------|
| `bazi.liuyue2026.bounds` | Worker 不回傳，前端用 `buildLiuyue2026Fallback` 生成；若 bazi 為 null 或 fallback 失敗則為空 |
| `#monthGrid` 是否存在 | 在 `#ws-liuyue` 內，`#system` 顯示後應可見 |
| 需滾動至 `#ws-liuyue` | 流月在八字五行下方，**需往下捲動**才看得到 |
| **修復** | `liuyue-month.js` 已加 fallback：當 `bounds` 為空時，呼叫 `buildLiuyue2026Fallback` 生成 12 月 |

### 2. 紫微五行比例（ziweiWxBars）不顯示

| 檢查項 | 說明 |
|--------|------|
| `computeZiweiElementRatios` | 從 `ziwei.mainStars` 統計 14 主星五行；僅主星有五行屬性 |
| `ziwei.mainStars` 結構 | Worker 回傳 `{ "命宮": ["紫微","天府"], "兄弟": [], ... }`，key 為繁體宮名 |
| 語系 | en-US 時星名為 emperor, advisor 等，需 `EN_STAR_TO_ZH_FOR_WUXING` 轉繁再查 `STAR_WUXING_MAP` |
| 全為 0 | 若無主星或主星名不匹配，ratios 全 0，bar 仍會畫出但很細 |
| **修復** | 計算失敗時，`data-renderer.js` 會用 `Calc.computeZiweiElementRatios(ziwei)` 單獨計算五行比例顯示 |

### 3. 共同前置：`#system` 是否顯示

- `#system` 預設 `class="hidden"`
- 計算完成後 `updateDashboardUI` 會移除 `hidden`
- 若計算流程出錯，`#system` 可能維持隱藏

### 4. 除錯指令（瀏覽器 Console）

```js
// 檢查 contract 與資料
console.log("ziwei.mainStars:", window.contract?.ziwei?.mainStars);
console.log("bazi.liuyue2026:", window.contract?.bazi?.liuyue2026);
console.log("ziweiScores:", window.ziweiScores);

// 手動計算五行比例
const ratios = window.Calc?.computeZiweiElementRatios?.(window.contract?.ziwei);
console.log("elementRatios:", ratios);

// 手動生成流月 fallback
const fb = window.Calc?.buildLiuyue2026Fallback?.(window.contract?.bazi);
console.log("liuyue fallback bounds:", fb?.bounds?.length);
```
