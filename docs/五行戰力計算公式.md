# 八字五行戰力計算公式

## 資料來源

- **四柱**：年柱、月柱、日柱、時柱（每柱 = 天干 + 地支）
- **知識庫**：`data/wuxing/stems.json`、`branches.json`、`month_branches.json`

---

## 一、表層結構（四柱天干）— surface

**定義**：只計天干 + 地支藏干，不計月令加權。

### 公式

```
v_raw[e] = Σ (天干貢獻) + Σ (地支藏干貢獻)
```

**天干貢獻**（預設權重 stem_w = 1.0）：
- 年干、月干、日干、時干各 +1
- 天干對應五行：甲乙→木、丙丁→火、戊己→土、庚辛→金、壬癸→水

**地支藏干貢獻**（預設權重 branch_w = 1.0）：
- 每個地支的 `hidden` 陣列：`[天干, 權重]`
- 貢獻 = branch_w × 權重

### 地支藏干表（branches.json）

| 地支 | 藏干 |
|------|------|
| 子 | 癸×1 |
| 丑 | 己×0.6, 癸×0.3, 辛×0.1 |
| 寅 | 甲×0.6, 丙×0.3, 戊×0.1 |
| 卯 | 乙×1 |
| 辰 | 戊×0.6, 乙×0.3, 癸×0.1 |
| 巳 | 丙×0.6, 庚×0.3, 戊×0.1 |
| 午 | 丁×0.7, 己×0.3 |
| 未 | 己×0.6, 丁×0.3, 乙×0.1 |
| 申 | 庚×0.6, 壬×0.3, 戊×0.1 |
| 酉 | 辛×1 |
| 戌 | 戊×0.6, 辛×0.3, 丁×0.1 |
| 亥 | 壬×0.7, 甲×0.3 |

### 範例（木=2, 火=1, 土=0, 金=0, 水=1）

假設四柱為：甲乙 / 丙丁 / 戊己 / 庚辛（示意）

- 木：甲+乙 = 2（或來自藏干如寅甲0.6、亥甲0.3 等）
- 火：1、土：0、金：0、水：1
- 顯示數值 = 原始加總，四捨五入保留 1 位小數

## 二、實戰結構（天干＋藏干＋月令加權）— strategic

**定義**：在 v_raw 基礎上，依「月柱地支」套用季節乘數。

### 公式

```
v_season[e] = v_raw[e] × seasonal_multiplier[月支][e]
```

**月令乘數表**（month_branches.json，依月支）：

| 月支 | 木 | 火 | 土 | 金 | 水 |
|------|-----|-----|-----|-----|-----|
| 寅、卯 | 1.3 | 1 | 0.9 | 0.95 | 1.1 |
| 辰、未、戌、丑 | 0.95 | 1.1 | 1.3 | 1 | 0.9 |
| 巳、午 | 0.95 | 1.3 | 1 | 0.9 | 1.1 |
| 申、酉 | 0.9 | 0.95 | 1.1 | 1.3 | 1 |
| 亥、子 | 1 | 0.9 | 0.95 | 1.1 | 1.3 |

### 顯示方式

- **條狀圖**：以 `maxStrategic = max(木,火,土,金,水)` 為分母，寬度 = `v_season[e] / maxStrategic × 100%`
- **數值**：`v_season[e]` 四捨五入保留 1 位小數

### 範例（木=1, 火=0.5, 土=0, 金=0, 水=0.5）

若月支為寅（木月）：
- 木：v_raw 木 × 1.3
- 火：×1、土：×0.9、金：×0.95、水：×1.1

若 v_raw 為 {木:0.77, 火:0.5, 土:0, 金:0, 水:0.45}，則 v_season 約為 {木:1.0, 火:0.5, 土:0, 金:0, 水:0.5}（木被 1.3 放大為最強）。

---

## 2.1 表層 vs 實戰 語意

| 維度 | 表層 (surface) | 實戰 (strategic) |
|------|----------------|------------------|
| 涵蓋 | 天干 + 藏干 | 天干 + 藏干 + 月令乘數 |
| 語意 | 角色名望、外表氣勢 | 核心戰力、實際能打程度 |
| 當令 | 不計 | 月支當令五行 ×1.3 |

## 三、Fallback（KB 載入失敗時）

當 `loadKB("/data/wuxing")` 失敗或 pillars 無法組出時：

**表層**：只計四柱天干，每個天干 +1
```
surface[五行] = 年干+月干+日干+時干 中屬該五行的個數
```

**實戰**：正規化到 0–1
```
strategic[五行] = surface[五行] / max(surface 各值)
maxStrategic = 1
```

### Fallback 觸發條件

1. `loadKB("/data/wuxing")` fetch 失敗（404、CORS、網路錯誤）
2. `buildPillarsFromBazi(bazi)` 回傳 null（缺 `bazi.display` 或 `bazi.year/month/day`）
3. `runPipeline` 拋錯

**診斷**：計算完成後，在 Console 輸入 `__wuxingFlowUsedFallback`，若為 `true` 表示走了 Fallback。  
測試 KB 載入：`await window.WuxingFlowPipeline?.checkWuxingKB()`。

---

## 四、數值差異的可能原因

1. **表層 vs 舊版**：舊版若只計天干，新版多了地支藏干，數值會變大、分布也會變。
2. **實戰 vs 表層**：實戰多了月令加權，月支當令的五行會被放大（×1.3），其他略減。
3. **Fallback vs Pipeline**：Fallback 只計天干，沒有藏干與月令，數值會較簡單、差異較大。
4. **KB 載入**：正式站若一度 404，會走 Fallback；修復後走 Pipeline，數值會不同。
