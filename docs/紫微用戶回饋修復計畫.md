## 一、問題分析

### 1. 地支寅宮位點擊無關聯解釋

**可能原因：**

| 原因 | 說明 | 優先級 |
|------|------|--------|
| A. 行動端 detailPanel 隱藏 | `max-width: 767px` 時 `#detailPanel { display: none }`，需靠 palaceSheet（Bottom Sheet）顯示。若 openPalaceSheet 未正確觸發或 sheet 內容空白，用戶看不到解釋 | 高 |
| B. 1280px 斷點不一致 | 宮位點擊時：`< 1280px` 呼叫 openPalaceSheet，`≥ 1280px` 滾動至 detailPanel。768–1279px 區間 detailPanel 可能仍隱藏（取決於 grid 設定），導致滾動目標不可見 | 中 |
| C. 寅格子的點擊區域 | 寅宮位對應 `grid-area: 4/1/5/2`（左下角），若中央區塊 `zw-center-block` 的 z-index 或邊界在某些 viewport 下覆蓋相鄰格子，可能阻擋點擊 | 低 |
| D. selectPalace 參數缺失 | palace-detail 需 ziwei、bazi、horoscope、dbContent 等。若 contract 未就緒或 dbContent 未載入，關聯解釋區可能為空 | 中 |

**驗證方式：**
- 在 寅 格子加 `console.log` 確認 click 事件有觸發
- 檢查 openPalaceSheet 是否被呼叫、palaceSheet 是否有 `open` class
- 確認 selectPalace 收到的 options 是否含 ziwei、dbContent

### 2. 大限年與局數沒有對齊

**根本原因：** `getStartAgeFromWuxingju` 一律回傳 2，未依五行局解析起始年齡。

**正確對照：**

| 五行局 | 起始年齡 | 大限轉換歲數 |
|--------|----------|--------------|
| 水二局 | 2 | 2, 12, 22, 32... |
| 木三局 | 3 | 3, 13, 23, 33... |
| 金四局 | 4 | 4, 14, 24, 34... |
| 土五局 | 5 | 5, 15, 25, 35... |
| 火六局 | 6 | 6, 16, 26, 36... |

**目前程式碼：**
- `baziCore.js`：`getStartAgeFromWuxingju` 固定 return 2
- `calc.js`：`buildSlotsFromZiwei` 用 `baseStartAge + k * 10`，若 baseStartAge 錯則全部錯
- `baziCore.js`：`getDecadalLimits` 也用 baseStartAge = 2

**影響：** 金四局、土五局、火六局用戶看到的「2–11、12–21...」與實際局數不符。

---

### 3. 小限位置跑掉

**根本原因：** `BaziCore.getHoroscopeFromAge` 回傳 `activeLimitPalaceName: null`，且註解寫「需要根據大限計算」。

**小限規則（簡化）：**
- 從命宮起，1 歲在命宮
- 陽男陰女順行，陰男陽女逆行
- 每年一宮

**目前狀況：**
- 後端 `compute/horoscope` API 若有回傳 `activeLimitPalaceName`，會覆寫本機值
- 本機 fallback（`getHoroscopeFromAge`）永遠回傳 null
- `buildSlotsFromZiwei` 的 fallback：`horoscope.yearlyIndex` 為整數時，用 `palaceOrder[yearlyIndex]`，但 `getYearlyIndexFromAge` 回傳的是「大限內第幾年」的索引，非小限宮位索引

**混淆點：**
- `yearlyIndex` 現為 `Math.floor((age - startAge) / 10)`，即「第幾個大限」
- 小限需「歲數對 12 取模」決定宮位，兩者不同

## 二、修復計畫

### Phase 1：大限年與局數對齊（優先，影響明顯）

1. **修改 `baziCore.getStartAgeFromWuxingju`**
   - 解析 wuxingju 字串（如 "金四局"、"水二局"）
   - 從中取出數字（二→2、三→3、四→4、五→5、六→6）
   - 回傳對應起始年齡

2. **修改 `baziCore.getDecadalLimits`**
   - 傳入 wuxingju，呼叫 `getStartAgeFromWuxingju` 取得 baseStartAge
   - 取代寫死的 baseStartAge = 2

3. **驗證 `calc.buildSlotsFromZiwei`**
   - 確認已使用正確 baseStartAge，無額外寫死

---

### Phase 2：小限位置修正

1. **實作 `getHoroscopeFromAge` 中的 activeLimitPalaceName**
   - 1 歲在命宮，依性別與命宮陰陽決定順行/逆行
   - `小限宮位索引 = (1 + direction * (age - 1) + 12*10) % 12`
   - 轉成宮位名稱：`palaceOrder[小限宮位索引]`

2. **釐清 yearlyIndex 語意**
   - 若需同時表達「大限索引」與「小限宮位」，考慮拆成 `decadalIndex` 與 `minorLimitPalaceIndex`
   - 避免用 yearlyIndex 同時表示兩者

---

### Phase 3：地支寅宮位點擊無解釋

1. **檢查 palaceSheet 與 openPalaceSheet**
   - 確認 `openPalaceSheet` 存在且會加上 `open` class
   - 確認 palaceDetailBody 內容由 selectPalace 正確填入

2. **統一斷點**
   - 若 768–1279px 仍隱藏 detailPanel，改為一致使用 palaceSheet，或調整 detailPanel 顯示條件

3. **加防呆與日誌**
   - selectPalace 若缺少必要資料，顯示「資料載入中」或明確錯誤訊息
   - 開發時可加 log 追蹤點擊與 openPalaceSheet 流程
