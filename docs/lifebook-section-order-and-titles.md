## 若畫面上 s04 仍在 s08 之後（順序不對）

**程式現狀**：  
- **Worker**（`worker/src/lifeBookTemplates.ts`）：`SECTION_ORDER` 已是 `s04, s02, s10, ...`；產出全書時（`index.ts`）依此順序寫入 `sections`。  
- **前端**（`src/lifebook-viewer/constants.ts`）：`SECTION_ORDER` 已改為 `s00, s03, s04, s02, s10, ...`，與 worker 一致。

**為何畫面上仍錯**：  
1. **Worker 未重新部署**：若命書是 worker 產出後回傳，需重新部署 worker 並**重新產生命書**（或清除已快取的舊結果）。  
2. **前端未重新部署**：若畫面是依前端的 `SECTION_ORDER` 渲染章節，需重新部署前端並強制重新載入（避免舊 bundle 快取）。  
3. **資料來源為舊版**：若章節來自資料庫或舊 API 回傳的既有內容，需以新順序重新生成並寫回。

**建議**：先重新部署 **worker**，再重新產出一本命書；若使用 lifebook-viewer 顯示，再確認前端已部署且無快取。
