# 使用統計與分析建議

> 目標：記錄每天使用人數（含未登入）、年齡／性別統計、地點；可考慮資料庫、GA4、SEO

---

## 一、現有資源盤點

| 來源 | 現有數據 | 限制 |
|------|----------|------|
| Cloudflare Web Analytics | 點擊、頁面瀏覽、獨立訪客、國別、裝置 | 需開通；無年齡／性別 |
| Cloudflare Workers Analytics | 請求數、錯誤率 | 無用戶屬性 |
| D1 (users, user_charts) | 登入者、出生日期、性別 | 僅登入用戶 |
| estimate_hour_logs | 推算時辰紀錄 | 有 user_id，偏特定功能 |

---

## 二、方案比較

### 2.1 資料庫（D1）自建

**適用**：要精準掌握「年齡／性別」、可與命盤綁定、可做長期分析。

| 優點 | 缺點 |
|------|------|
| 年齡、性別直接來自表單，最準 | 僅記錄「有送出命盤」的人 |
| 可與現有 D1 整合、匯出分析 | 需自己設計 schema、API、隱私處理 |
| 不需依賴第三方服務 | 寫入量大時需考慮 D1 寫入限制 |

**建議 schema（匿名使用紀錄）**

```sql
-- 每次「算出命盤」時寫入一筆（不論是否登入）
CREATE TABLE usage_logs (
  id TEXT PRIMARY KEY,
  -- 匿名辨識（同 session 短時間內可去重）
  session_id TEXT,
  -- 使用者資料（來自表單，不涉及個資）
  birth_year INTEGER,
  gender TEXT,           -- male/female/other
  language TEXT,         -- zh-TW, en-US
  -- 時空
  created_at TEXT DEFAULT (datetime('now')),
  -- 地理（可從 Cloudflare 請求頭取得，僅國家/地區）
  country TEXT,
  region TEXT,
  -- 隱私：不存 IP，可選 hash(IP) 做粗略去重
  ip_hash TEXT
);

CREATE INDEX idx_usage_created ON usage_logs(created_at);
CREATE INDEX idx_usage_year_gender ON usage_logs(birth_year, gender);
```

**統計維度**：日活、週活、年齡區間分布、性別比、語言、地區。

---

### 2.2 Google Analytics 4 (GA4)

**適用**：要看流量、頁面行為、漏斗、SEO 效果；可與 Cloudflare 互補。

| 優點 | 缺點 |
|------|------|
| 免費、功能完整 | 需掛 JavaScript，略影響載入 |
| 內建事件、轉換、受眾 | 年齡／性別來自 GA 模型推估，不是真實表單值 |
| 可接 Search Console 做 SEO | 需接受 cookie、GDPR 同意（歐洲） |
| 儀表板現成、可匯出 | 中國大陸難以使用 |

**建議用法**

1. **GA4 財產**：建立一個 GA4 財產，綁定網域。
2. **自訂事件**：
   - `calculation_complete`：命盤計算完成（可帶 `birth_year` 區間、`gender`）
   - `chart_viewed`：進入結果頁
3. **自訂維度**：`birth_year_range`、`gender`、`language`。
4. 隱私：用 `gtag('consent', ...)` 處理同意；若主要用戶在台灣，可簡化。

**SEO 關聯**：GA4 + Search Console 可看搜尋查詢、點擊率、排名。

---

### 2.3 Cloudflare Web Analytics

**適用**：不想掛第三方 script、要輕量、重視隱私。

| 優點 | 缺點 |
|------|------|
| 無需 cookie、符合 GDPR | 無法取得年齡／性別 |
| 與 Pages / Workers 同平台 | 分析功能不如 GA4 細 |
| 點擊、訪客、國家、裝置現成 | 無法自訂事件（或功能較少） |

**建議**：作為「流量與使用人數」基礎數據，與 D1 的年齡／性別統計互相印證。

## 三、實作優先順序

| 優先 | 項目 | 成本 | 效益 |
|------|------|------|------|
| 1 | 開啟 Cloudflare Web Analytics | 低 | 立刻有 UV、國家、裝置 |
| 2 | D1 `usage_logs` + Worker API | 中 | 年齡、性別真實數據 |
| 3 | GA4 基本安裝 + 自訂事件 | 低 | 行為、轉換、SEO 整合 |
| 4 | 後台統計儀表板（admin） | 中 | 可視化、匯出 |

---

## 四、年齡／性別紀錄時機

建議在 **Worker 的 compute/all 成功回傳後** 觸發：

1. 前端：收到成功後發送 `POST /api/log-usage`（或併入 compute 回傳前的邏輯）。
2. 前端 payload：`{ birth_year, gender, language }`（來自表單）。
3. Worker：驗證格式、從 `cf-ipcountry` 取國家、寫入 D1。
4. 可選：同一 `session_id` 或 `ip_hash` 在 24 小時內只計一次（避免重複送出）。

## 五、SEO 建議

| 項目 | 說明 |
|------|------|
| **meta / title** | 每頁獨立的 title、description、og:image |
| **structured data** | 可在首頁加 `WebApplication` 或 `SoftwareApplication` JSON-LD |
| **sitemap** | 靜態或動態 sitemap.xml 給 Search Console |
| **hreflang** | zh-TW / en-US 若為不同 URL，加 hreflang |
| **Core Web Vitals** | Cloudflare / GA4 可監測 LCP、CLS、FID |
| **Search Console** | 綁定網域、提交 sitemap、定期看查詢與點擊 |

## 六、隱私與合規

- **年齡／性別**：非個資，但建議在隱私政策中說明蒐集目的與用途。
- **IP**：若只存 hash 做去重，風險較低；不建議存原始 IP。
- **Cloudflare**：已在處理流量，國別等為其內建功能。
- **GA4**：若用戶含歐洲，需 consent mode；台灣可先簡化。
- **D1**：資料在 Cloudflare，需在隱私政策註明「使用 Cloudflare 服務」。

---

## 七、總結建議

1. **先用 Cloudflare Web Analytics**：零開發成本，先掌握流量與使用人數。
2. **新增 D1 `usage_logs`**：在 compute 流程寫入年齡、性別、語言，做真實統計。
3. **再上 GA4**：補足行為分析與 SEO 整合。
4. **後台儀表板**：可查 D1 的日／週統計、年齡與性別分布；Cloudflare 與 GA4 用官方 dashboard。

## 十、GA4 已實作項目

1. **index.html**：GA4 gtag 腳本（依 `window.GA_MEASUREMENT_ID` 動態載入）
2. **config.js**：`GA_MEASUREMENT_ID` 與 HTML 同步
3. **ui.js**：命盤計算成功後發送 `calculation_complete` 事件

**GA4 自訂事件參數**：

| 事件名稱 | 觸發時機 | 參數 |
|----------|----------|------|
| `calculation_complete` | 命盤計算成功後 | `birth_year_range`, `gender`, `language` |

**設定步驟**：

1. 到 [Google Analytics](https://analytics.google.com/) 建立 GA4 財產
2. 取得 Measurement ID（格式 `G-XXXXXXXXXX`）
3. 在 `index.html` 將 `window.GA_MEASUREMENT_ID = 'G-XXXXXXXXXX'` 替換為你的 ID
4. 留空或非 `G-` 開頭則不載入 gtag，不影響本地開發
