# 部署步驟（一步一步）

## 步驟 1：取得 Cloudflare API Token

1. 開啟瀏覽器，前往 https://dash.cloudflare.com/
2. 登入你的 Cloudflare 帳號
3. 點右上角頭像 → **My Profile**
4. 左側選單點 **API Tokens**
5. 點 **Create Token** 按鈕
6. 選擇範本 **Edit Cloudflare Workers**，或點 **Use template**
7. 權限確認（預設應包含 Workers Scripts Edit）
8. 點 **Continue to summary** → **Create Token**
9. **複製產生的 token**（只顯示一次，請存好）

---

## 步驟 2：建立 .env.deploy 檔案

1. 在專案根目錄開啟終端，執行：

```bash
cd /Users/bill/bazi-project
cp .env.deploy.example .env.deploy
```

2. 用編輯器開啟 `.env.deploy`，把 `你的_token` 改成步驟 1 複製的實際 token
3. 儲存檔案

---

## 步驟 3：執行部署指令

在終端機（Terminal 或 Cursor 內建的終端）輸入：

```bash
cd /Users/bill/bazi-project
```

按 Enter，再輸入：

```bash
source .env.deploy && npm run deploy
```

按 Enter，等待部署完成。

---

## 步驟 4：確認結果

部署成功後會看到：
- Worker 部署到 `bazi-api.billeetw.workers.dev`
- Pages 部署到 Cloudflare Pages（如 `*.bazi-d1p.pages.dev`）

---

## 若中途出錯

| 錯誤 | 處理 |
|------|------|
| `CLOUDFLARE_API_TOKEN` 未設定 | 檢查 `.env.deploy` 路徑是否正確、是否在專案根目錄 |
| `source: no such file` | 確認檔名是 `.env.deploy`（開頭有 dot） |
| 權限不足 / EPERM | 關閉沙盒，或在本機 Terminal 直接執行 |
