## 使用方式

### 在代碼中調用

```javascript
// 表層（角色名望）
window.Calc.getBoyanBoard(bazi.wuxing?.surface, "surface");
renderFiveElementComment("surfaceWxComment", bazi.wuxing?.surface, "surface");

// 實戰（核心戰力）
window.Calc.getBoyanBoard(bazi.wuxing?.strategic, "strategic");
renderFiveElementComment("strategicWxComment", bazi.wuxing?.strategic, "strategic");
```

### 資料庫匯入

`wuxing-descriptions.json` 可以直接匯入到資料庫的 `ui_copy_texts` 表，使用以下 key：
- `表層木`, `表層火`, `表層土`, `表層金`, `表層水`
- `實戰木`, `實戰火`, `實戰土`, `實戰金`, `實戰水`

## 文案特色

### 表層（角色名望）- 描述「他人看到的你」
- 木：外在成長模組 - 展現開展性與人際連結
- 火：外在亮度模組 - 展現吸睛度與反應速度
- 土：外在穩定模組 - 展現可靠形象與持久力
- 金：外在界線模組 - 展現規則感與判斷力
- 水：外在溝通模組 - 展現資訊流通與柔性魅力

### 實戰（核心戰力）- 描述「你真正在依靠什麼能力做事」
- 木：策略模組 - 展開能力與起始動作
- 火：推進力模組 - 執行動能與爆發力
- 土：耐力模組 - 沉澱力與累積能力
- 金：決斷力模組 - 切割能力與邏輯紀律
- 水：洞察模組 - 感知與資料處理

## 語氣特點

1. **冷靜、有溫度**：像人生工程師在陪你讀自己的系統
2. **完整結構**：每段包含系統意義、當前狀態、最佳使用方式、風險提示、行動建議
3. **不使用恐嚇詞**：避免「劫、刃、凶、報應、破財」等玄學恐嚇詞
4. **具體行動建議**：每段結尾都有「今年，你需要練習...」的具體指引

## 向後兼容

所有函數都保持向後兼容，預設使用「實戰」描述。未傳入 `kind` 參數時，會自動使用 `"strategic"`。
